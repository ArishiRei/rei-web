# 修复 Stylelint 样式规范问题
Date: 2025-12-23 17:05

## Actions
1. **排除构建产物**: 创建 `.stylelintignore` 文件，忽略 `dist/`, `.output/`, `.nuxt/`, `node_modules/` 目录，防止对构建生成的压缩 CSS 进行检查。
2. **自动修复样式**: 执行 `pnpm lint:style --fix` 修复了 90+ 处规范问题，包括：
    - **颜色格式**: 将 `rgba()` 统一为现代 `rgb()` 语法，并将 alpha 值从浮点数改为百分比（如 `0.5` -> `50%`）。
    - **Hex 缩写**: 将 `#ffffff` 缩写为 `#fff`。
    - **媒体查询**: 将旧版 `min-width/max-width` 语法转换为现代的范围表示法（如 `width >= 768px`）。
    - **代码格式**: 修复了 At-rule 前的空行问题。
3. **验证流程**: 运行 `pnpm lint` 确认 JS 和 Style 检查全部通过。

## 问题与解决
- **问题**: `pnpm lint` 报告了大量 `dist/` 目录下的错误。
- **解决**: 通过配置 `.stylelintignore` 解决了该问题，确保只检查源码。
- **问题**: 部分现代 CSS 语法（如范围表示法）可能在旧版浏览器中不兼容。
- **解决**: 本项目目前优先遵循 Stylelint 的现代规范建议，后续若有兼容性需求，可通过 PostCSS 进行自动降级处理。
