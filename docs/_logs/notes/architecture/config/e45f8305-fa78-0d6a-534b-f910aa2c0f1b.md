# 组件自动导入配置与规范更新
Date: 2025-12-22 20:00
## Actions
1. 无依赖安装
2. 开发逻辑与步骤：
   - **配置更新**:
     - 修改 `nuxt.config.ts` 的 `components` 选项。
     - 显式定义 `app/components/` 下的 `base`, `domain`, `layout`, `ui` 目录，并设置 `pathPrefix: false`。
     - 效果：消除组件自动导入时的目录前缀（例如 `BaseReiButton` -> `ReiButton`）。
   - **规范更新**:
     - 更新 `docs/prompt.md`，明确组件封装的路径策略和命名规范，强调无需使用目录前缀。
3. 问题与解决：
   - 解决了 Nuxt 默认组件自动导入会将目录名作为前缀的问题，使组件使用名更加简洁且符合 `Rei` 前缀规范。
