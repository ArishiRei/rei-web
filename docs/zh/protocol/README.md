# 开发协议

本目录包含 Nuxt 4 项目的核心开发协议，从原始的 `docs/prompt.md` 文件中提取并模块化。协议被组织成专注、可维护的文档，定义了项目的标准、原则和工作流程。

## 协议结构

### 核心文档

- **[核心原则](./principles.md)** - 基本开发原则和架构哲学
- **[架构标准](./architecture.md)** - 架构标准和目录结构指南  
- **[编码规范](./coding-standards.md)** - 代码风格、质量要求和代码检查规则
- **[工作流程](./workflow.md)** - 开发流程、Git 工作流和提交标准
- **[质量验证](./validation.md)** - 质量门禁、DoD 检查清单和验证标准

## 导航指南

### 新开发者
1. 从 [核心原则](./principles.md) 开始了解项目哲学
2. 查看 [架构标准](./architecture.md) 了解项目结构
3. 学习 [编码规范](./coding-standards.md) 获取实现指南
4. 遵循 [工作流程](./workflow.md) 进行开发流程
5. 使用 [质量验证](./validation.md) 确保完成前的质量

### 经验丰富的团队成员
- **快速参考**: 使用下方的交叉引用直接跳转到特定章节
- **更新**: 修改标准时，更新相关协议文档并维护交叉引用
- **新模式**: 在适当的协议章节中记录新的架构模式

## 交叉引用系统

### 架构标准 → 编码规范
- [架构标准](./architecture.md) 中的目录结构指南由 [编码规范](./coding-standards.md) 中的代码检查规则强制执行
- 组件组织模式引用两个文档

### 工作流程 → 质量验证  
- [工作流程](./workflow.md) 中的 Git 提交标准由 [质量验证](./validation.md) 中的检查清单验证
- 构建流程引用两个文档的质量门禁

### 核心原则 → 所有文档
- [核心原则](./principles.md) 中的核心原则在所有其他协议文档中实现
- 每个文档章节都应与基本原则保持一致

## 协议维护

### 更新协议文档
1. 确定需要修改的协议文档
2. 更新特定文档同时维护交叉引用
3. 验证更改不与其他协议章节冲突
4. 如需要新的交叉引用，更新此 README

### 添加新标准
1. 确定新内容的适当协议文档
2. 按照文档现有结构添加新标准
3. 创建到其他文档相关章节的交叉引用
4. 如新标准需要强制执行，更新验证标准
